How the algorithms work

Both versions compare the same underlying team statistics — they differ in how those stats become a final prediction.

Algo V1 — point system

Each stat is normalized into a 1–10 style score, summed into a total, then mapped to win probability with a league-specific curve.

  • Season record differential
  • Home/away performance gap
  • Recent 10-game form
  • Average scoring margin

Algo V2 — probability system

Each factor gets its own best-fit curve from backtests, producing percentage contributions that are averaged into a final win chance.

  • More accurate in original NBA backtests
  • NHL adds home/away win streak signal
  • Outputs American moneyline equivalents

Data & backtesting

The repo ships with years of ESPN-scraped box scores plus OddsPortal lines for historical validation.

  • NBA/NHL data through 2017
  • MLB data through 2016
  • CLI backtester still available

About this project

This modern web front-end wraps James Quintero's open-source Sports Odds Algorithms. The core Python engine remains intact; the site exposes its predictions through a FastAPI service and a responsive UI for exploration and demos.

Live schedule scraping via ESPN may require updates to the legacy scraper. The demo uses the bundled historical CSV datasets, which are sufficient to reproduce documented examples such as Portland @ Golden State in the 2017 NBA playoffs.
